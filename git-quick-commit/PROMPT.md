Write a single git commit message for the staged changes shown below.

## Output rules

- Output ONLY the commit message text. No code fences, no preamble, no sign-off, no commentary.
- Do not run commands or use tools. Use only the commit log and diff provided below.

## Content rules

- Keep it minimal. Prefer a single subject line. Add a body ONLY if the change is not self-explanatory from the subject, and then only to explain WHY, never to restate the diff.
- Match the exact style of the recent commits below: their prefix/scope convention, capitalization, mood, and length. Bias toward the most recent.
- Subject: imperative mood, no trailing period, aim for <=50 characters.
- Describe what the change accomplishes as a whole, not a file-by-file list.
- No attribution, Co-Authored-By, or "Generated with" trailers.
